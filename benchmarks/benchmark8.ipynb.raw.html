Warning¶

Submission Guidelines¶

Please use the upload form to upload your results.

In addition to that specified, further data to upload can include a Youtube video, images of the nuclei at different times, or the entire phase field variable at different times. These are not required, but will help others view your work.

Part (a)¶

Run three cases with starting seeds with $r_0 = r^*$, $r_0 = 0.99 r^*$, and $r_0 = 1.01 r^*$. Follow the time evolution of the system for $t = 100$ units. The following data should be collected for upload as frequently as possible.

  • The solid fraction in the domain (the integral $\int_V \phi dV / \int_V dV$)

  • The free energy, $\mathcal{F}$

One PFHub upload is required with two data contributions for each of the three simulations (for a total of six data contributions). The data contributions should be named as solid_fraction_1 and free_energy_1, solid_fraction_2 and free_energy_2 as well as solid_fraction_3 and free_energy_3 where _1, _2 and _3 refers to $r_0 = r^*$, $r_0 = 0.99 r^*$, and $r_0 = 1.01 r^*$, respectively. The data contributions should be stored in either CSV of JSON format. The columns name for time and quantity need to be identified correctly on the upload form for each data contribution. The data should probably be stored in three separate files for each simulation and each file with three columns for time, free energy and solid fraction.

Further data to upload can include a Youtube video, images of the nucleus at different times or the entire phase field variable at different times. These are not required, but will help others view your work.

Part (b)¶

Follow the time evolution of the system for $t = 400$ units. The following data should be collected for upload as frequently as possible.

  • The solid fraction in the domain (the integral $\int_V \phi dV / \int_V dV$)

  • The free energy, $\mathcal{F}$

Two data contributions are required and should be named as solid_fraction and free_energy, and stored in either CSV or JSON format. The results can be stored in two separate files or one single file as long as each column name for time and quantity is identified on the upload form. In addition, the phase field snapshot at time $t=80$ is required. The data should be named as image and stored in PNG or TIFF format.

Part (c)¶

Follow the time evolution of the system for $t = 600$ units. The following data should be collected for upload as frequently as possible.

  • The solid fraction in the domain (the integral $\int_V \phi dV / \int_V dV$)

  • The free energy, $\mathcal{F}$

Two data contributions are required and should be named as solid_fraction and free_energy, and stored in either CSV or JSON format. The results can be stored in two separate files or one single file as long as each column name for time and quantity is identified on the upload form. In addition, the phase field snapshot at time $t=100$ is required. The data should be named as image and stored in PNG or TIFF format.

Part (d)¶

Run three cases with undercooling $\Delta f_0 = \Delta f_0^*$, $\Delta f_0 = 0.98 \Delta f_0^*$, and $\Delta f_0 = 1.02 \Delta f_0^*$. Follow the time evolution of the system for $t = 5000$ units. The following data should be collected for upload as frequently as possible.

  • The solid fraction in the domain (the integral $\int_V \phi dV / \int_V dV$)

One PFHub upload is required with one data contribution for each of the three simulations (for a total of three data contributions). These should be named as solid_fraction_1, solid_fraction_2 and solid_fraction_3 and stored in either CSV or JSON format. In addition, the phase field snapshot at time $t=5000$ is required. The data should be named as image and stored in PNG or TIFF format.
